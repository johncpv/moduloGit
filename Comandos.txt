Configurando ambiente GIT

*Informar usuário que esta logado
git config --global user.name "John Marcus"

*Informar email do usuario logado
git config --global user.email "johncpv@gmail.com"

*Informar editor utilizado
git config --global core.editor vscode

*Visualizar configurações.
git config --list  (visualização geral)
git config user.name (visualizar nome)

CONCEITO BRANCH (VERSÃO DIFERENTES DO REPOSITORIO)

Branch MASTER --> Versão oficial do repositório.

CONCEITO COMIT (EXECUTA ALTERAÇÕES NO REPOSITORIO)

ao dar um comit vc envia para o seu repositorio oficial (GIT) os arquivos que vc estava alterando localmente.

ADICIONANDO ARQUIVOS NO GIT

git status (mostra o status dos arquivos na pasta local e no GIT)
git add -A (para enviar todos arquivos que estão na pasta mas não estão sendo monitorados pelo GIT)
git commit -am "Primeiro commit do projeto" (confirmando subida das alterações.)
git log (exibe as ultimas ações na branch que esta.)
git branch (lista todos os branch do projetos) 
git reset --soft||mixer||hard (comando para voltar estado)
    -soft (Volta antes do commit. Mantem os arquivos para futuras verificações.) ***O mais usado
    -mixer (Volta antes do ADD. Mantem os arquivos para futuras verificações. )
    -hard (Volta do local indicado. Apaga os arquivos pra frente.)

CONCEITO BRANCH

é um controle de "história" a parte do principal.

git branch nomebranch (Criar novos branch)
git checkout nomebranch (mudar para o branch sinalizado)
git checkout (Voltar, pode ser um branch, um arquivo)
git checkout -- (quando coloca -- é que esta esperando um nome e não um comando)
